DisViz

DisViz is a visualization tool for studying executions of distributed systems. It produces a time-space diagram by parsing a JSON log file:

DisViz is built off and extends ShiViz. See their README for a description of their tool, and their Wiki for an in-depth introduction.

See Section 3.3 (page 33) of my thesis for a description of DisViz, or just clone this repo and run it. The best way to learn the tool is to use it!

GoVector is a logging library that can be used to generate DisViz-compatible JSON logs. See this repo for a description of how to use these tools together.

Installation

To run this project, you'll need Node.js. You can download it from the Node.js website.

After that, just clone this repo and you're good to go!

Usage

DisViz uses a Node process as a server, which parses the entire log file and communicates with the browser to show a subset of the log file at a given time.

To start the server, run npm start from the cloned repo.

Paste the filepath of the JSON log file you want to visualize. The format should be two initial newline characters followed by one JSON log entry per line. Each log must have the fields 'processId', 'message', and 'VCString'. You can hide processes, search for text matches and communication patterns, and inspect individual log events:

Motivation

The reason for making a fork of the original ShiViz was to make it easier to interact with large log files, and to add flexibility when adding information to individual log statements.

DisViz was built as part of my MEng, see Chapter 3 (page 28) of the thesis for a thorough description of the tool. See Chapter 6 (page 58) for examples of debugging Raft bugs with this tool; Section 6.3 walks through debugging a snapshotting bug that I never found when I first took 6.5840!

Modifying DisViz

I have not sufficiently tested DisViz beyond the capabilities required for my MEng. There are many opinionated pieces of the implementation, and I'm sure there are bugs lurking around.

If you find a bug or want a feature that doesn't exist, feel free to create an issue or make a pull request!
